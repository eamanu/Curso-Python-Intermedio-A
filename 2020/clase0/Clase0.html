Funciones¶

Podemos crear funciones, usando la palabra reservada def y podemos setear parmetros dentro del parntesis. Una funcin puede no tener parmetros.

In [12]:
def print_mensaje():
    print("Hola a todos!!!") 

def print_hello(nombre):
    print(f"hola {nombre}")


print_mensaje()    
print_hello("Emmanuel")

hola Emmanuel

Cuando se envan argumentos a una funcin, y estos se reciben los parmetros en el orden que fueron defindos, estos son argumentos por posicin

In [13]:
def foo(a, b):
    return a * b

res = foo(2, 3)
print(res)

Tambin se puede enviar los parmetros con sus nombres, ya no estamos obligados a llamar a la funcin respetando el orden con el que fueron definidos

In [14]:

POO¶

Este es un paradigma de programacin que cambia la forma de procesar datos. Con la POO ya no tenemos una secuencia de comandos sino que tenemos ahora entidades u objetos, que tienen un comportamiento y caractersticas, y permite la interaccin entre objetos, por ejmplo mediante el envo de mensajes. Las tres caractersticas principales de la POO son: Encapsulacin, Polimorfismo y Herencia.

Clases¶

Representa una abstraccin una generalizacin. La clase es una encapsulacin porque constituye una encapsula los datos que conforman los objetos como los procedimientos que permiten manipularlos.

Objetos¶

Cada objeto tiene una identidad, un tipo y un valor. Un objeto representa alguna entidad de la vida real, y que pertenecen al problema con el que nos estamos enfrentando, y con el que podemos interactuar.

Módulos¶

En Python, cada uno de nuestros archivos .py se denominan mdulos. Estos mdulos, a la vez, pueden formar parte de paquetes. Un paquete, es una carpeta que contiene archivos .py. Pero, para que una carpeta pueda ser considerada un paquete, debe contener un archivo de inicio llamado init.py. Este archivo, no necesita contener ninguna instruccin. De hecho, puede estar completamente vaco.
